# Visual Flow 作成・運用ルール

このフォルダ（visual-flow）は、Web アプリケーションの画面仕様を管理するための設計ドキュメントを格納する。Visual Flow 方式（Sony Computer Entertainment の PS4 Shell 設計で用いられた方式を参考）に基づく。

---

## 目的

- 画面仕様の一元管理
- 画面遷移の可視化
- 新規画面追加・変更時の手順の明確化

---

## フォルダ構成

```
.sdd/visual-flow/
├── conventions.md          # 本ファイル（作成・運用ルール）
├── transition-flow.md      # 画面遷移を集約したファイル
├── v_xxx.md                # 各画面の設計書（並列配置）
└── v_xxx/                  # 補足用（画像等。必要時のみ作成）
```

---

## 命名規則

| 項目 | 規則 |
|------|------|
| 画面設計書 | `v_xxx.md`（xxx は任意、スネークケース推奨） |
| 補足フォルダ | `v_xxx/`（v_xxx.md に収まらない画像・補足資料を格納） |

---

## ファイル名の原則

- **ファイル名には極力階層構造を持ち込まない**（例: `v_admin_project_edit` ではなく、親画面と並列に `v_admin_project_detail`）
- 同じ機能の画面を使い回すことが多いため、フラットな命名を推奨
- `admin` や `issue` などのプレフィックスはそのままでよい（今後 Issue 関連画面が増える想定）

---

## 配置ルール

- **一画面一ファイル**の原則
- **並列配置**：フォルダ内も階層構造にしない。一覧と詳細のように URL 上は親子でも、`v_admin_projects.md` と `v_admin_project_detail.md` は同じ階層に並べる
- **画面遷移は transition-flow.md に集約**する

---

## v_xxx フォルダの用途

- v_xxx.md に記載しきれない補足資料
- 画像（*.md から参照する画像はこのフォルダに置く）

---

## 新規画面追加時の手順

1. `v_xxx.md` を追加（テンプレートに従って作成）
2. [transition-flow.md](transition-flow.md) の対応表・遷移図を更新
3. 必要に応じて `v_xxx/` フォルダを作成し、画像等を配置
